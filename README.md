# SpacePixels 🌌

**SpacePixels** is a powerful, user-friendly astronomical image (FITS) analysis tool designed specifically for amateur astronomers to easily detect moving objects such as asteroids, comets, and artificial satellites.

Originally built as a pre-processing utility to align and prepare files for other software, SpacePixels now features its own **fully integrated, multi-threaded automatic object detection engine**. Simply load a set of aligned and calibrated images (generated by stacking software such as DeepSkyStacker or Siril), and let the pipeline find the transients for you!

Please check the [SpacePixels Wiki](https://github.com/ppissias/SpacePixels/wiki)
## ✨ Key Features

### 🚀 Automated Moving Object Detection
* **Multi-Threaded Engine:** Rapidly analyzes FITS sequence files utilizing all available CPU cores.
* **Smart Shape Classification:** Distinguishes between slow-moving point sources (asteroids) and fast-moving targets (satellite/meteor streaks).
* **Stationary Star Purification:** Automatically builds a master star map to purge stationary objects and isolate true transients.
* **Intelligent Quality Control:** Automatically detects and ignores bad frames caused by passing clouds, wind blur (FWHM spikes), or mount tracking errors.

### 📊 Comprehensive HTML Reporting
* **Visual Output:** Automatically generates a dark-themed HTML diagnostic report for every detection run.
* **Animated Track Summaries:** Creates perfectly cropped, animated GIFs of detected targets (both Object-Centric and Star-Centric views) so you can visually verify transients instantly.

### 👁️ Manual Inspection & Processing
* **Blink Animation:** Manually stretch and blink a set of FITS images to visually hunt for moving objects.
* **Real-Time Stretching:** Apply custom, non-linear stretch algorithms with real-time visual previews.
* **Batch Processing:** Easily batch-convert color FITS files to monochrome, and batch-stretch large datasets.

### 🗺️ Plate Solving
* **ASTAP Integration:** Fast, local plate solving using your local ASTAP installation.
* **Astrometry.net Integration:** Online fallback solving via Nova Astrometry.net.
* **WCS Application:** Solve a single image and apply the celestial coordinates to the FITS header of the entire aligned sequence.

## 🔭 Why I Created SpacePixels: The Opposite of Stacking

In modern astrophotography, our primary goal is almost always to create a single, deep, noise-free image of a deep-sky object. To do this, we take dozens or hundreds of individual sub-exposures and run them through stacking software.

Those stacking algorithms (like Sigma Clipping) are incredibly good at doing one specific thing: **throwing away anything that changes between frames**.

But what exactly are we throwing away? Those "rejected pixels" and "artifacts" are often real, dynamic events happening above our heads—asteroids, comets, meteors, artificial satellites, and unknown transients. By stacking, we are literally erasing the active universe to look at the static one.

**SpacePixels is the exact opposite of a stacking tool.** It was built to mine your discarded data. Instead of averaging out movement to hide it, SpacePixels actively hunts for movement across your sub-exposures to reveal the hidden transients you captured while you weren't looking.

### ⚠️ Crucial Requirement: Aligned & Calibrated Inputs
Because SpacePixels looks for tiny pixel movements between frames, it needs the background starfield to be perfectly stationary.

Therefore, **SpacePixels requires perfectly aligned (registered) and calibrated FITS images as its input.** SpacePixels does *not* align the images for you. Instead, it is designed to use the intermediate files generated by your existing stacking software. Simply take the aligned, calibrated outputs from software like **DeepSkyStacker**, **Siril**, or **PixInsight**—the exact files generated right before the final master stack is created—and load them into SpacePixels.
